The design calls here
Same canvas as the GM report — the light "findings dossier." Agent app is dark and immersive; the GM report and this Console share the sober, printable document canvas. (Confirms the provisional "Console follows GM-report canvas" note.)
Access mirrors the GM report. Ctrl + Alt + R + PIN — the same chord as the GM's Ctrl+Alt+A, with R for Researcher; auto-locks on exit.
Six lenses, canonical, integrity-first. Aligned to Researcher-Value-Model v0.2: Integrity · Validation · Agent Activity · App Health · GM Activity · Iteration. Coverage/convergence folds under Integrity; the console leads with the researcher's real question — is the instrument sound?
Validation is the North Star. The structured GM-reaction capture (non-obvious? / would act?) is the researcher's key signal — a research activity, not a product feature; the GM never sees it; simulation-only for the pilot.
Badges, never names. Agent activity is per-badge and anonymous — engagement can be monitored without identifying a person; no output ever ties an answer to a badge.
GM activity is open-log only. Opens and dwell time, never what the GM read or did — enough to know the findings landed, nothing invasive.
Fixed from v2: retired "Stag" badge → real final-set coins; emoji → line-art; dark canvas → dossier; added the R-key access line. Plain-language: lens labels are researcher-facing (English-fluent), so the ESL bar applies less here than on agent screens.
