The design calls here
1 · Downtime reassures first. The agent's instinct is "did I lose my work?" — so the answer ("your answers are safe," "saves on its own") comes before the mechanics. Plain words, one clear action: Try again.
2 · Two different audiences. The offline screen is agent-facing and calm; the config-error screen is really for whoever set up the device — so it names the fix ("ask the researcher") and carries a small technical line the agent can ignore.
3 · Never blames the agent. Both say the system isn't ready — never "you did something wrong."
4 · Offline-tolerant by design. As a PWA, reads are held locally and sync when the connection returns — the copy promises exactly that, no more.
